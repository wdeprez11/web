My Dad always dissauded me in learning C and told me to focus on Java, I'm sure he was just trying to do what was in my best interest, but realistically I think it was best for me to learn C/C++ to become a better programmer.

The most difficult part learning C (for me), was getting used to using pointers and references. In this article I hope to explain the proper cases in which to use either -- and what pointers and references are.

Regular variables

So what do we know about Regular variables? First, we know they store the address and value of the specific static types (E.g. int, float, char, etc..)

Pointers

So what is a pointer, and why are they special? A pointer... points! A pointer is a variable that points to an address of memory of the specific type of pointer.

Visualizing pointers

In Memory Table below, you can see we have a variable int *p, this variable points to the memory address 0003. We have a second variable int x which stores the value 25 and the memory address 0003

It's important to notice the distinction of the asterisk in int *p as its explaining that it's a pointer of type int -- that it is capable of pointer to the address of an int value.

Declaring pointers

There are several ways to declare a pointer and it's important to make the distinction because you may end up declaring incorrectly and mess up your entire program.

int* p: this declaration is the most common in C++ code, it states that p is a pointer to an int. One problem with using this in your code is: whilst declaring you may encounter an error when attempting declare multiple pointers instead of an integer and then a pointer.

Example error

The following includes an error in which you accidentally declare two pointers instead of a regular int and a pointer. However, you can't actually declare these on the same line like the following example.


int main() {
    int* n = 5, q = &n;
    return 0;
}

int *p: more common in C code, it denotes the pointer bound to the p.

This really should only be used when declaring a pointer and int in the same line, but it's not the best practice.

int * p: evaluates the same as int *p, it's really just a compromise between the two.

It's best to stick with the C++ method of int* ptr as opposed to anything else.
